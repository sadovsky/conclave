# Conclave v0.2 Plan

This document specifies the two primary features for Conclave v0.2:

1. **Expanded DSL** — richer control flow and computation in the `want` block
2. **Goal Modules** — content-addressed goal imports and a module registry

Both features preserve the core invariant: determinism is structural, not aspirational. Every new construct lowers to a deterministic Plan IR. Every import is a hash commitment.

---

## Part 1: Expanded DSL

v0.1 has one control structure: `map`. v0.2 adds four more.

### `if/else` — conditional branching

```conclave
goal ClassifyAndProcess(urls: List<Url>) -> Json {
  want {
    map urls as url {
      let html = fetch(url);
      if is_article(html) {
        let text = extract_text(html);
        emit summarize(text);
      } else {
        emit skip(url);
      }
    }
    return assemble_json(collected);
  }
}
```

- `is_article` is a capability returning a `Bool` type
- Both branches must have compatible `emit` types
- Lowerer expands to a `ConditionalNode` in Plan IR with two subgraph edges: `branch_true` and `branch_false`
- The scheduler dispatches the condition first, then dispatches the appropriate branch
- Determinism: branch selection is a function of capability output, which is recorded in the trace

### `reduce` — sequential left-fold

```conclave
goal MergePages(urls: List<Url>) -> Html {
  want {
    reduce urls as url into acc {
      let page = fetch(url);
      acc = merge_html(acc, page);
    }
    return acc;
  }
}
```

- `into acc` names the accumulator; initial value is the zero value for the return type
- Ordered strictly by list index — no parallelism
- Lowerer expands to a chain of `MergeNode` entries, one per list item (like `map` but with explicit ordering and data threading)
- Unlike `map`, `reduce` does not emit into `collected` — it produces a single value

### `pure` — inline intrinsic computation

```conclave
goal CountWords(urls: List<Url>) -> Json {
  want {
    map urls as url {
      let html = fetch(url);
      let count = pure { word_count_bytes(html) };
      emit count;
    }
    return assemble_json(collected);
  }
}
```

- `pure { ... }` contains intrinsic calls only — no capabilities, no IO
- The lowerer emits an `IntrinsicCall` node (same as existing `assemble_json` handling)
- Purpose: avoid a subprocess round-trip for simple transformations
- The set of allowed intrinsics in `pure` blocks is the same as the `intrinsic` declaration namespace

### Multiple goals per file

```conclave
version 0.1;

goal FetchPage(url: Url) -> Html {
  want {
    return fetch(url);
  }
}

goal ProcessPage(html: Html) -> String {
  want {
    let text = extract_text(html);
    return summarize(text);
  }
}
```

- Each goal lowers to its own independent Plan IR
- `conclave lower file.conclave --goal FetchPage` selects by name
- `conclave lower file.conclave` (no `--goal`) lowers all goals and emits a JSON array
- Goals in the same file may not call each other directly — use Goal Modules for composition
- Each goal's `plan_ir_hash` is independent; changing one goal does not affect the other's hash

## Part 2: Goal Modules

Goal modules make goals first-class artifacts: content-addressed, versioned by hash, importable by other goals.

### DSL addition

New top-level declaration (before `goal`):

```conclave
import FetchAndExtract: sha256:<64 hex chars>;
```

Grammar:
```
import_decl := "import" IDENT ":" HASH ";"
```

- `IDENT` becomes a callable name inside `want` blocks, used exactly like a capability
- `HASH` must be a valid `sha256:` hash referencing a Plan IR in the module cache
- Multiple imports are allowed; names must be unique within a file

Example usage:

```conclave
version 0.1;

import FetchAndExtract: sha256:abc123...;

capability summarize: summarize(String) -> String;
intrinsic assemble_json: assemble_json(List<String>) -> Json;

goal SummarizePages(urls: List<Url>) -> Json {
  want {
    map urls as url {
      let text = FetchAndExtract(url);
      emit summarize(text);
    }
    return assemble_json(collected);
  }
}
```

### Lowerer changes

**Resolution phase** (before the `want` block is lowered):

1. For each `import` declaration, look up the hash in the module cache
2. Parse the sub-goal's Plan IR JSON
3. Validate that the sub-goal's input/output types match the call site types
4. Store the resolved sub-goal IR keyed by import name

**Expansion phase** (during `want` block lowering):

When the lowerer encounters a call to an imported name (e.g., `FetchAndExtract(url)`):

1. Inline the sub-goal's Plan IR as a subgraph at the call site
2. Wire the call site's input value to the sub-goal's input port
3. Wire the sub-goal's output port to the binding variable
4. Tag every inlined node with `subgraph_id: sha256:...` (the import hash)

The resulting Plan IR is fully expanded — no runtime module resolution needed.

**Files to modify:**
- `crates/conclave-lang/src/ast.rs` — add `ImportDecl` AST node
- `crates/conclave-lang/src/lexer.rs` — add `import` keyword token
- `crates/conclave-lang/src/parser.rs` — parse `import_decl` at top level
- `crates/conclave-lang/src/normalize.rs` — validate import hash format, sort imports
- `crates/conclave-lang/src/lower.rs` — resolve and expand imports before lowering `want`

### Plan IR changes

**`crates/conclave-ir/src/lib.rs`:**

```rust
pub struct IrNode {
    // ... existing fields ...
    pub subgraph_id: Option<String>,  // sha256 hash of the imported module, if any
}

pub struct PlanIr {
    // ... existing fields ...
    pub imports: BTreeMap<String, String>,  // import name → plan_ir_hash
}
```

`subgraph_id` enables:
- Trace attribution: "this node executed as part of `FetchAndExtract` v`sha256:abc123...`"
- Partial auditing: filter trace by subgraph
- Future: incremental re-sealing when only top-level bindings change

### Manifest changes

**`crates/conclave-manifest/src/lib.rs`:**

```rust
pub struct Manifest {
    // ... existing fields ...
    pub module_bindings: BTreeMap<String, String>,  // import name → plan_ir_hash
}
```

**Seal changes (`crates/conclave-seal/src/lib.rs`):**

- Walk the full expanded Plan IR including subgraphs
- Collect all `CapabilityCall` nodes, including those with `subgraph_id` set
- Require a `capability_bindings` entry for every capability in the expanded graph
- Validate `module_bindings` entries match the import hashes in the Plan IR header

This means Agent B's manifest must explicitly bind all capabilities that `FetchAndExtract` uses — even though Agent B didn't declare them. This is intentional: it makes the transitive dependency graph explicit and auditable.

### Module cache

**Location:**
- macOS: `~/Library/Caches/conclave/modules/`
- Linux: `~/.cache/conclave/modules/`

**Format:** `<sha256_hex>.json` — canonical Plan IR JSON (same format as `plan_ir.json`)

### New CLI commands

```
conclave module publish <source.conclave> [--goal NAME]
```
Lower the goal, write the Plan IR to the module cache, print the `plan_ir_hash`.

```
conclave module install <sha256_hash> [--registry URL]
```
Fetch a module by hash from a registry and install into the local cache.

```
conclave module list
```
List all modules in the local cache (hash + first 40 chars of canonical JSON).

```
conclave module inspect <sha256_hash>
```
Print the Plan IR and capability signatures for a cached module.

## Implementation order

1. Plan IR: add `subgraph_id` and `imports` fields (backward-compatible, both optional)
2. Manifest: add `module_bindings` field (optional, defaults to empty)
3. DSL: lexer + parser + normalize for `import_decl`
4. Module cache: read/write utilities (mirrors capability store pattern)
5. Lowerer: resolution and expansion of imports
6. Seal: transitive capability validation
7. CLI: `conclave module` subcommands
8. Expanded DSL: `if/else`, `reduce`, `pure`, multi-goal (independent of modules)

Steps 1–7 are sequential (each depends on the previous). Step 8 is independent and can proceed in parallel once Plan IR changes land.

## Verification

After full implementation:

```bash
# Agent A publishes a module
conclave module publish fetch_and_extract.conclave
# → module_hash: sha256:abc123...

# Agent B writes a program importing it
cat > summarize.conclave << 'EOF'
version 0.1;
import FetchAndExtract: sha256:abc123...;
capability summarize: summarize(String) -> String;
intrinsic assemble_json: assemble_json(List<String>) -> Json;
goal SummarizePages(urls: List<Url>) -> Json {
  want {
    map urls as url {
      let text = FetchAndExtract(url);
      emit summarize(text);
    }
    return assemble_json(collected);
  }
}
EOF

# Lower — expanded Plan IR includes FetchAndExtract subgraph
conclave lower summarize.conclave --url-count 2 --output plan_ir.json
# → plan_ir_hash commits to full expanded graph

# Inspect: nodes from FetchAndExtract carry subgraph_id
conclave inspect plan_ir.json

# Seal — must bind all transitive capabilities (fetch, extract_text, summarize)
conclave seal --plan plan_ir.json --manifest manifest.json --output sealed.json

# Pack + run as normal
conclave pack --runtime ./target/release/conclave \
  --plan plan_ir.json --manifest sealed.json --output artifact.cnclv
conclave run artifact.cnclv \
  --urls "https://example.com,https://anthropic.com" \
  --trace-out trace.json --mode live

# Changing fetch_and_extract.conclave produces a new hash
# Agent B's import still pins sha256:abc123... → lower fails with hash mismatch
# Agent B must explicitly update import to accept the new version
```
